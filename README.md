# Burmese Emoji Search

![AI-Assisted](https://img.shields.io/badge/Built%20With-AI%20Assisted-6200ea?style=for-the-badge&logo=dependabot&logoColor=white)

A Burmese and English emoji search experience that blends fast lexical matching with semantic vector search. The app keeps ranking in the browser for instant results while delegating query embedding generation to a lightweight backend proxy and a Hugging Face Space.

Developed by **Heinko**.

## Overview

This project helps Myanmar-language users find emojis by exact meaning, partial syllable matches, and semantic intent. A search for `"ဝမ်းနည်း"` or `"sad"` can still surface crying-face emojis even when the keywords are not a perfect literal match.

For a deeper system walkthrough, see [search_architecture.md](/Users/heink/v0-burmese-emoji-search-su/search_architecture.md).

## Features

- Hybrid search that combines Burmese syllable matching with multilingual semantic embeddings
- Client-side ranking against a precomputed emoji vector index for fast response time
- Query embeddings generated by a Hugging Face Space instead of a heavy Vercel serverless function
- Unicode and CLDR-based data pipeline with local Burmese keyword overrides

## Architecture

At runtime, the browser keeps the emoji dataset in memory, calls `/api/embed` for the current query embedding, and then scores everything locally.

```mermaid
flowchart LR
    U["User types query"] --> C["Next.js client"]
    C --> L["Lexical scorer<br/>exact, substring, syllable"]
    C --> A["/api/embed on Vercel"]
    A --> S["Hugging Face Space"]
    S --> V["384d query vector"]
    V --> C
    C --> R["Cosine similarity vs emoji vectors"]
    L --> M["Merged score"]
    R --> M
    M --> O["Top ranked emojis"]
```

The data pipeline is separate from the runtime path:

```mermaid
flowchart TD
    U["Unicode emoji data"] --> B["update-emoji.ts"]
    C["CLDR Myanmar annotations"] --> B
    O["data/locales/my.csv overrides"] --> B
    B --> E["Transformers.js embedding generation"]
    E --> J["public/data/emoji/emoji-index-my.json"]
    J --> R["Runtime browser search"]
```

## Runtime Components

1. `public/data/emoji/emoji-index-my.json` stores emoji metadata plus precomputed 384-dimensional vectors.
2. `hooks/use-semantic-search.ts` computes lexical scores and asks `/api/embed` for the query vector when semantic mode is enabled.
3. `app/api/embed/route.ts` is a small proxy route that forwards embedding requests to the public Hugging Face Space.
4. `hf-space-embed-service/` contains the Docker-based Space service that loads `intfloat/multilingual-e5-small`.

## Updating Emoji Data

To rebuild the Burmese emoji index:

```bash
npm run update-emoji
```

That script will:

1. Download Unicode emoji definitions
2. Download CLDR Myanmar annotations
3. Merge `data/locales/my.csv` overrides
4. Generate embeddings for each emoji entry
5. Write the final JSON index into `public/data/emoji/`

## Hugging Face Space

The embedding service source lives in `hf-space-embed-service/`. It is designed for a Docker-based Hugging Face Space and exposes:

- `GET /health`
- `GET /embed?q=...`

By default, the app uses the deployed public Space URL baked into `/api/embed`. You can override it later with `EMBEDDING_SERVICE_URL` if needed.
